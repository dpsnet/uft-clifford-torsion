# TNN 发育式智能 Roadmap

## 概述

基于「能力驱动生长」核心机制，设计从简单到复杂、从单技能到多技能、从具身到离身的完整发育路径。

---

## 选项 A：长程记忆任务（测试生长的必要性）

### 目标
验证「容量瓶颈 → 触发生长 → 解决瓶颈」的完整闭环。

### 任务设计

**任务名称**：间隔复制任务 (Interval Copy Task)

**输入输出**：
- 输入：START [noise×gap] PATTERN [noise×gap] END
- 输出：复制 PATTERN
- 关键：必须记住跨越 gap 的 PATTERN

**难度递进**：
```
Level 1: gap=4   (序列长度 10)
Level 2: gap=8   (序列长度 18)  
Level 3: gap=16  (序列长度 34)
Level 4: gap=32  (序列长度 66)
Level 5: gap=64  (序列长度 130)
```

### 实施阶段

#### Phase A1: 基础验证（1-2天）
**目标**：验证2层模型无法处理长间隔

- [ ] 实现间隔复制数据生成器
- [ ] 实现静态2层TNN基线
- [ ] 测试不同间隔长度的准确率曲线
- [ ] **预期结果**：间隔>16时准确率<50%

#### Phase A2: 生长机制验证（2-3天）
**目标**：验证生长解决容量瓶颈

- [ ] 实现能力驱动的生长检查器
- [ ] 设置生长阈值：准确率>80%持续100步 → 加1层
- [ ] 训练并监控生长触发点
- [ ] **预期结果**：
  - gap=8时需要3层
  - gap=16时需要4层
  - gap=32时需要5层

#### Phase A3: 对照实验（1-2天）
**目标**：证明「生长」优于「预训练大模型」

- [ ] 实验组：生长模型（2层→5层）
- [ ] 对照组1：静态5层（从头训练）
- [ ] 对照组2：静态2层（无法学会）
- [ ] **对比指标**：
  - 收敛速度（生长模型更快）
  - 最终性能（生长模型更稳定）
  - 参数效率（生长模型用更少参数达到同等性能）

#### Phase A4: 谱维协同（2-3天）
**目标**：验证谱维与层数协同生长

- [ ] 实现谱维自适应解锁
- [ ] 设置：低谱维=2.5（短程），高谱维=4.5（长程）
- [ ] 观察谱维随任务复杂度自动提升
- [ ] **预期结果**：处理gap=64时，谱维自动提升至4.0+

## 选项 B：多技能解锁（功能模块化）

### 目标
从单一技能开始，逐步解锁感知、记忆、语言、社交四层能力。

### 技能树设计

```
                    ┌─────────────┐
                    │   社交期    │  ← 多智能体协调
                    │ (Social)    │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │   语言期    │  ← 符号通信
                    │ (Language)  │
                    └──────┬──────┘
                           │
            ┌──────────────┼──────────────┐
            │              │              │
     ┌──────▼──────┐ ┌─────▼──────┐ ┌─────▼──────┐
     │   记忆期    │ │   感知期   │ │   运动期   │
     │  (Memory)   │ │ (Perception)│ │  (Motor)   │
     └─────────────┘ └────────────┘ └────────────┘
                           │
                    ┌──────▼──────┐
                    │   胚胎期    │  ← 原始感觉-运动映射
                    │  (Embryo)   │
                    └─────────────┘
```

### 实施阶段

#### Phase B1: 胚胎期（感觉-运动映射）（2-3天）
**目标**：最简单的输入-输出映射

- [ ] 任务：光传感器 → 转向运动
- [ ] 网络：2层，16维内部空间
- [ ] 成功标准：趋光性/避光性行为稳定

#### Phase B2: 记忆期解锁（3-4天）
**目标**：添加记忆模块，学会延迟反应

**解锁条件**：
- 胚胎期任务准确率 > 90%
- 连续 200 步达标

**新增模块**：
- 短期记忆缓冲区（循环连接）
- 记忆读写门控

**新任务**：
- 延迟匹配任务：看到信号 → 等待 → 选择正确方向
- 成功标准：能记住信号并延迟 5-10 步正确反应

#### Phase B3: 语言期解锁（4-5天）
**目标**：添加符号处理能力

**解锁条件**：
- 记忆期任务准确率 > 85%
- 能稳定使用记忆模块

**新增模块**：
- 符号嵌入层
- 序列预测头
- 注意力机制扩展

**新任务**：
- 指令跟随：「红色→左转」「蓝色→右转」
- 序列学习：简单语法规则
- 成功标准：能理解3-5个符号指令

#### Phase B4: 社交期解锁（5-7天）
**目标**：多智能体协调

**解锁条件**：
- 语言期任务准确率 > 80%
- 能稳定使用符号系统

**新增模块**：
- 社交感知层（检测其他智能体状态）
- 通信通道（信息素/信号）
- 合作/竞争决策模块

**新任务**：
- 简单合作：共同搬运食物
- 角色分工：领导者/跟随者
- 成功标准：2个智能体能完成需要配合的任务

#### Phase B5: 整合测试（3-4天）
**目标**：验证四层技能能同时使用

- [ ] 复杂场景：感知 + 记忆 + 语言 + 社交同时启用
- [ ] 场景示例：
  - 智能体A发出「危险」信号
  - 智能体B听到信号，记住位置
  - 稍后引导智能体C避开该区域

## 选项 C：TNN皮层+脑干融合（终极架构）

### 目标
将「具身智能」（生物TNN）和「离身智能」（Transformer TNN）融合为统一架构。

### 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                     皮层系统 (Cortex)                        │
│                  TNN-Transformer                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │  语言理解    │  │  推理规划    │  │  抽象概念    │      │
│  │  (Language)  │  │ (Reasoning)  │  │  (Concepts)  │      │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘      │
│         │                 │                 │              │
│         └─────────────────┼─────────────────┘              │
│                           │                                │
│                  ┌────────▼────────┐                       │
│                  │   全局工作空间   │                       │
│                  │  (Global Workspace)                     │
│                  └────────┬────────┘                       │
└───────────────────────────┼─────────────────────────────────┘
                            │
┌───────────────────────────┼─────────────────────────────────┐
│                     丘脑接口 (Thalamus)                      │
│              扭转场耦合层 (Torsion Coupling)                 │
│                                                              │
│   上行：感觉信息 → 符号表征                                  │
│   下行：高层指令 → 运动控制                                  │
│                                                              │
│   ┌──────────────────────────────────────────────────────┐  │
│   │  互反空间 ↔ 内部空间 双向映射                          │  │
│   │  低维感觉 ←→ 高维抽象                                  │  │
│   └──────────────────────────────────────────────────────┘  │
└───────────────────────────┼─────────────────────────────────┘
                            │
┌───────────────────────────┼─────────────────────────────────┐
│                    脑干系统 (Brainstem)                      │
│                    TNN-BioModel                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │   感觉输入   │  │   反射回路   │  │   运动输出   │      │
│  │  (Sensors)   │  │  (Reflexes)  │  │  (Motors)    │      │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘      │
│         │                 │                 │              │
│         └─────────────────┼─────────────────┘              │
│                           │                                │
│                  ┌────────▼────────┐                       │
│                  │   内部空间动态    │                       │
│                  │ (Internal Space) │                       │
│                  └──────────────────┘                       │
└─────────────────────────────────────────────────────────────┘
```

### 实施阶段

#### Phase C1: 皮层系统搭建（5-7天）
**目标**：建立高层认知模块

- [ ] 基于修复版 TNN-Transformer
- [ ] 配置：512维隐藏层，8层，8头，4M参数
- [ ] 训练任务：
  - 自然语言指令理解
  - 简单逻辑推理
  - 目标规划（多步任务分解）

#### Phase C2: 脑干系统搭建（3-4天）
**目标**：建立低层感知-行动模块

- [ ] 复用 TNN小鼠/TNN果蝇架构
- [ ] 配置：64维内部空间，4层，500K参数
- [ ] 连接虚拟环境：
  - 视觉输入（模拟摄像头）
  - 运动输出（轮速/关节角度）
  - 本体感觉（位置/速度）

#### Phase C3: 丘脑接口实现（核心难点）（7-10天）
**目标**：实现双向扭转场耦合

**技术挑战**：
1. **维度不匹配**：皮层512维 vs 脑干64维
2. **时间尺度不匹配**：皮层慢思考 vs 脑干快反射
3. **信息类型不匹配**：符号 vs 感觉运动

**解决方案**：

**上行通路（感觉→认知）**：
```
脑干内部空间 [64维] 
    ↓ 投影层
互反空间 [128维]
    ↓ 扭转场编码
丘脑表征 [256维]
    ↓ 注意力选择
皮层输入 [512维]
```

**下行通路（认知→行动）**：
```
皮层指令 [512维]
    ↓ 目标分解
子目标序列 [256维]
    ↓ 扭转场解码
运动原语 [128维]
    ↓ 投影层
脑干控制 [64维]
```

**扭转场耦合机制**：
- 使用扭转场作为「共同货币」统一表示
- 皮层的高维抽象通过扭转压缩传递到低层
- 低层的感觉通过扭转展开传递到高层

#### Phase C4: 分层训练（5-7天）
**目标**：分别训练后联合微调

1. **预训练阶段**：
   - 皮层单独训练（语言/推理）
   - 脑干单独训练（感知/运动）

2. **接口训练阶段**：
   - 冻结皮层和脑干权重
   - 只训练丘脑接口层
   - 任务：皮层指令 → 脑干执行

3. **联合微调阶段**：
   - 全网络端到端训练
   - 任务：复杂指令跟随

#### Phase C5: 整合测试（3-5天）
**目标**：验证「高层规划 + 低层执行」协同

**测试场景**：
1. **指令**：「去厨房拿苹果」
2. **皮层处理**：
   - 解析指令 → 目标：找到苹果
   - 规划路径 → 子目标序列
3. **丘脑传递**：
   - 将「去厨房」分解为运动指令
4. **脑干执行**：
   - 视觉导航
   - 避障
   - 抓取动作

**成功标准**：
- 能理解10+种自然语言指令
- 能将指令转化为连续动作序列
- 遇到障碍时能重新规划（皮层介入）

## 优先级与依赖关系

### 推荐执行顺序

```
选项 A (长程记忆)
    │
    ▼ (验证生长机制可靠)
选项 B (多技能解锁)
    │
    ▼ (验证模块化生长可行)
选项 C (皮层+脑干融合)
```

**理由**：
1. A是B的基础（技能解锁本质上是模块生长）
2. B是C的预演（皮层+脑干 = 最高级的技能组合）
3. 风险逐级增加，先做低风险验证

### 并行可能性

- A和B的Phase 1-2 可以并行
- C的Phase 1-2 可以与B的后阶段并行
- 但C的Phase 3（接口）必须在A、B都成功后进行

## 风险评估

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 生长机制不稳定 | A/B/C全失败 | 先做A的最小验证 |
| 接口训练不收敛 | C失败 | 设计多种接口方案 |
| 时间成本超预期 | 项目延期 | 设置检查点，及时止损 |
