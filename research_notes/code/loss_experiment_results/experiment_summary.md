# TNN损失函数调优实验结果 - 2026-03-19

## 实验目的
测试不同损失函数配置对0.90M参数TNN-Transformer扭转场能量稳定性的影响

## 测试配置

1. **baseline** - 原始配置（正则化系数0.0001）
2. **strong_reg** - 10倍强正则化（系数0.001）
3. **min_energy_5** - 强正则化 + 最小能量约束5.0
4. **entropy_bonus** - 强正则化 + 熵奖励（鼓励多样性）

## 实验结果

| 配置 | 最终损失 | 最终扭转场能量 | 最终谱维 | 能量稳定性 |
|------|----------|----------------|----------|------------|
| baseline | 6.9180 | 0.0000 | 6.19 | ❌ 归零 |
| strong_reg | 6.9150 | 0.0000 | 6.21 | ❌ 归零 |
| **min_energy_5** | 6.9287 | **4.9800** | 6.21 | ✅ 稳定 |
| entropy_bonus | 6.8395 | 0.0000 | 6.20 | ❌ 归零 |

## 关键发现

1. **单纯增加正则化系数无效**：即使10倍强正则化（0.001），扭转场能量仍然在100步内归零

2. **最小能量约束是唯一有效策略**：min_energy_5配置成功将扭转场能量稳定在4.98左右，且基础损失与其他配置相当（6.93 vs 6.91-6.92）

3. **熵奖励对生成多样性有帮助**：entropy_bonus配置虽然能量归零，但基础损失最低（6.84），说明熵奖励确实有助于提升生成质量

4. **谱维在所有配置中都稳定在6.2左右**，不受扭转场能量影响

## 推荐配置

**对于1.39M参数TNN-Transformer最大化存储能力：**

```python
torsion_coef = 0.001        # 强正则化
torsion_min_energy = 5.0   # 最小能量约束（关键！）
```

这个配置可以：
- 保持扭转场能量稳定在5.0左右
- 基础损失与其他配置相当
- 防止模型退化为普通Transformer
